互联网是网络的支柱，以这种技术为基础使网络成为可能。作为基础，互联网是把电脑互相连接起来的一个巨大网络。
互联网是一种基础的技术，它允许我们把成千上万的电脑连接在一起。在这些电脑中，有 一些电脑（我们称之为网络服务器（Web servers））可以发送一些浏览器可以理解的信息。互联网是基础设施，网络是建立在这种基础设施之上的服务。值得注意的是，一些其他服务运行在互联网之上，比如邮箱和IRC.

互联网是如何工作的

互联网地址

  • 因为互联网是一个全球计算机网络，每台连接到互联网的计算机必须有一个独特的地址。互联网地址以n.n.n.n. n的形式出现，其中 n 必须是 0 - 255 的号码。此地址称为 IP 地址
  • 如果您通过互联网服务提供商 （ISP） 连接到互联网，则通常会在拨入会话期间分配一个临时 IP 地址。如果您从局域网 （LAN） 连接到 Internet，您的计算机可能有永久 IP 地址，或者它可能从 DHCP（动态主机配置协议）服务器获得临时 IP 地址。无论如何，如果您连接到互联网，您的计算机有一个唯一的 IP 地址

协议堆栈和数据包

因此，您的计算机连接到互联网，并具有唯一的地址。它如何与连接到互联网的其他计算机”交谈”？

假设您的 IP 地址为 1.2.3.4，并且您想向计算机发送消息 5.6.7.8。
您要发送的消息是”你好电脑 5.6.7.8！
显然，消息必须通过连接计算机到互联网的任何类型的电线传输。
假设您从家里拨入了ISP，并且消息必须通过电话线传输。
因此，该消息必须从字母文本转换为电子信号，通过互联网传输，然后翻译回字母文本。
这是如何实现的？通过使用协议堆栈

由于使用了两种主要通信协议,互联网上使用的协议堆栈被称为 TCP/IP 协议堆栈

如果我们按照”你好电脑5.6.7.8！” 的消息从我们的计算机到IP地址5.6.7.8的计算机路径，它会发生这样的事情：

  • 消息将从计算机上协议堆栈的顶部开始，并朝下工作。
  • 如果要发送的消息很长，则消息传递的每个堆栈层可能会将消息分解为较小的数据块。这是因为通过Internet（和大多数计算机网络）发送的数据是以可管理的块发送的。在互联网上，这些数据块被称为数据包。
  • 数据包将通过应用层并继续到 TCP 层。每个数据包都分配一个端口编号。端口稍后将进行解释，但足以说明许多程序可能正在使用 TCP/IP 堆栈并发送消息。我们需要知道目的地计算机上的哪个程序需要接收消息，因为它将在特定端口上收听。
  • 通过 TCP 层后，数据包将进入 IP 层。这是每个数据包接收其目的地地址的地方，5.6.7.8。
  • 现在，我们的消息包有一个端口号码和一个IP地址，他们准备通过互联网发送。硬件层负责将包含我们消息字母文本的包转为电子信号并通过电话线传输。
  • 在电话线的另一端，您的ISP与互联网有直接连接。ISP路由器检查每个包中的目的地地址，并确定将其发送到何处。通常，数据包的下一站是另一个路由器。稍后将更多有关路由器和互联网基础设施的信息。
  • 最终，数据包到达计算机 5.6.7.8。在这里，数据包从目的地计算机的 TCP/IP 堆栈的底部开始，向上工作。
  • 当数据包向上穿过堆栈时，从数据包中剥离了发送计算机堆栈添加的所有路由数据（如 IP 地址和端口号）。
  • 当数据到达堆栈顶部时，数据包已重新组装成原始表单，”你好计算机 5.6.7.8！

互联网基础设施

互联网骨干网由许多相互连接的大型网络组成。这些大型网络称为网络服务提供商或NSP。一些大型 NSP 是 UUNet、瑟夫网、IBM、BBN 行星、冲刺网、PSINet 以及其他 NSP。这些网络相互对等以交换数据包流量。每个 NSP 需要连接到三个网络接入点或NAP。在 NAP 中，分组流量可能会从一个 NSP 的骨干跳到另一个 NSP 的骨干。NSP 在大都会地区交易所或 MAE 的MAE也相互连接，其用途与 NAP 相同，但属于私有。NAP 是最初的互联网互连点。NAP 和 MAE 都被称为互联网交换点或IX.NSP 也向较小的网络（如 ISP 和较小的带宽提供商）销售带宽。下面是显示此分层基础结构的图片。

互联网路由等级

那么，数据包如何在互联网上找到它们的方式呢？连接到互联网的每台计算机都知道其他计算机的位置吗？数据包只是将”广播”到互联网上的每台计算机吗？
问题的回答都是”不”。没有计算机知道任何其他计算机的位置，并且数据包不会发送到每台计算机。用于将数据包传到目的地的信息包含在连接到 Internet 的每个路由器保存的路由表中。

路由器是数据包开关路由器通常在网络之间连接到它们之间的路由包。每个路由器都知道它的子网络以及他们使用的 IP 地址。路由器通常不知道”上面”的IP地址是什么。


当数据包到达路由器时，路由器检查源计算机上的 IP 协议层放置的 IP 地址。路由器检查路由表。如果发现包含 IP 地址的网络，则将数据包发送到该网络。如果找不到包含 IP 地址的网络，则路由器会在默认路线上发送数据包，通常将主干等级向上发送到下一个路由器。希望下一个路由器将知道在哪里发送包。如果没有，则再次向上路由数据包，直到它到达 NSP 骨干。连接到 NSP 骨干站的路由器拥有最大的路由表，在这里，数据包将被路由到正确的主干，在那里它将开始通过越来越小的网络”向下”的旅程，直到它找到目的地。

域名和DNS

如果您不知道要连接到的计算机的 IP 地址，该怎么办？如果您需要访问称为”www.anothercomputer.com"的 Web 服务器，该怎么办？您的网络浏览器如何知道这台计算机在互联网上的所在地？
问题的答案是域名服务或DNS

DNS 是一个分布式数据库，可跟踪计算机名称及其在互联网上的相应 IP 地址。
许多连接到 Internet 的计算机承载了 DNS 数据库的一部分，以及允许其他人访问它的软件。这些计算机称为 DNS 服务器。没有DNS服务器包含整个数据库：它们只包含它的子集。如果 DNS 服务器不包含其他计算机请求的域名，DNS 服务器将请求的计算机重新定向到其他 DNS 服务器。

域名服务的结构类似于 IP 路由等级。请求名称解析的计算机将重新定向”向上”分层，直到发现 DNS 服务器可以解决请求中的域名。图6说明了等级的一部分。树的顶部是域根。一些较旧的、更常见的域出现在顶部附近。没有显示的是世界各地的众多 DNS 服务器，这些服务器构成了其余的等级制度。
设置互联网连接（例如用于 Windows 中的局域网或拨号网络）时，通常指定一个主服务器和一台或多个辅助 DNS 服务器作为安装的一部分。这样，任何需要域名解析的互联网应用程序都能够正常工作。例如，当您将网址输入您的 Web 浏览器时，浏览器首先连接到您的主 DNS 服务器。在获取您输入的域名的 IP 地址后，浏览器将连接到目标计算机并请求您想要的网页。

互联网协议

互联网运行需要许多通信协议。其中包括 TCP 和 IP 协议、路由协议、中等访问控制协议、应用级别协议等。
以下部分描述了互联网上一些更重要和常用的协议。
首先讨论更高级别的协议，然后是较低级别的协议。

应用协议：HTTP和WWW

互联网上最常用的服务之一是万维网（WWW）。
使网络工作的应用程序协议是超文本传输协议或HTTP。
当在 Web 浏览器中键入 URL 时，将发生这种情况：

  • 如果 URL 包含域名，浏览器首先连接到域名服务器并检索 Web 服务器的相应 IP 地址。
  • Web 浏览器连接到 Web 服务器，并为所需的网页发送 HTTP 请求（通过协议堆栈）。
  • Web 服务器会收到请求并检查所需的页面。如果页面存在，则网络服务器会发送该页面。如果服务器找不到所请求的页面，它将发送 HTTP 404 错误消息。（404 的意思是”找不到页面”，因为任何上网的人可能都知道。
  • Web 浏览器接收回页面并关闭连接。
  • 然后，浏览器分析页面并查找完成网页所需的其他页面元素。这些通常包括图像、苹果等。
  • 对于所需的每个元素，浏览器会针对每个元素向服务器提供额外的连接和 HTTP 请求。
  • 当浏览器完成加载所有图像、苹果等内容后，页面将完全加载到浏览器窗口中。

应用协议：SMTP 和 Electronic Mail

Electronic Mail使用一种称为”简单邮件传输协议”或”SMTP”的应用程序级别协议。
SMTP 也是基于文本的协议，但与 HTTP 不同，SMTP 以连接为导向。SMTP也比高频波复杂。SMTP 中的命令和考虑比 HTTP 中的命令和考虑要多得多。

当您打开邮件客户端以阅读您的电子邮件时，通常会出现这种情况：

  • 邮件客户端（网景邮件、莲花笔记、微软展望等）打开连接到其默认邮件服务器的连接。邮件服务器的 IP 地址或域名通常是在安装邮件客户端时设置的。
  • 邮件服务器将始终传输第一条消息来识别自己。
  • 客户端将发送 SMTP HELO 命令，服务器将响应该命令并发送 250 OK 消息。
  • 根据客户端是否正在检查邮件、发送邮件等，将向服务器发送相应的 SMTP 命令，服务器将做出相应的响应。
  • 此请求/响应事务将继续，直到客户发送 SMTP 退出命令。然后，服务器将说再见，连接将被关闭。

传输控制协议 TCP

协议堆栈中的应用层下是 TCP 层。当应用程序打开 Internet 上连接到另一台计算机的连接时，它们发送的消息（使用特定的应用程序层协议）会从堆栈中传递到 TCP 层。
TCP 负责将应用程序协议路由到目的地计算机上的正确应用程序。
为此，使用端口编号。端口可以被认为是每台计算机上的单独通道。
例如，您可以在阅读电子邮件时上网。这是因为这两个应用程序（Web浏览器和邮件客户端）使用了不同的端口号码。
当数据包到达计算机并向上设置协议堆栈时，TCP 层会根据端口编号决定哪个应用程序接收数据包。

TCP 的工作原理是这样的：

  • 当 TCP 层从上面接收应用层协议数据时，它会将其细分为可管理的”块”，然后向每个”块”添加带有特定TCP信息的TCP头。TCP头中包含的信息包括数据需要发送到的应用程序的端口编号。
  • 当TCP层从其下方的IP层接收数据包时，TCP层会从数据包中剥离TCP头数据，必要时进行一些数据重建，然后使用从TCP头取的端口编号将数据发送到正确的应用程序。
  • 这就是 TCP 如何将通过协议堆栈移动的数据路由到正确的应用程序。

TCP不是文本协议。
TCP 是一种以连接为导向、可靠、小节流服务。
基于连接的意味着使用 TCP 的两个应用程序必须在交换数据之前首先建立连接。
TCP 是可靠的，因为对于收到的每一个数据包，都会向发件人发送确认信，以确认递送。
TCP 还包括一个检查器，用于检查接收的数据。

网际协议 IP

与 TCP 不同，IP 是一个不可靠、无连接的协议。
IP不关心数据包是否到达目的地。
IP 也不知道连接和端口编号。
IP 的工作是将数据包发送和路由到其他计算机。IP 数据包是独立的实体，可能出现问题或根本不正常。
TCP 的工作是确保数据包到达并按正确的顺序到达。
关于 IP 与 TCP 的唯一共同点是它接收数据并将自己的 IP 标题信息添加到 TCP 数据中的方式。

什么是HTTP/HTTPS

HTTP 是 Web 浏览器和 Web 服务器用来通过互联网相互通信的协议
它是一个应用程序级别协议，因为它位于协议堆栈中的 TCP 层顶部，并被特定应用程序用于相互交谈。
在这种情况下，应用程序是 Web 浏览器和 Web 服务器。
HTTP 是一个基于文本的无连接协议。

  • 客户端（Web 浏览器）向 Web 服务器发送有关网页和图像等 Web 元素的请求。
  • 请求由服务器服务后，客户端和服务器之间的互联网连接将断开。
  • 对于每个请求，必须进行新的连接。大多数协议都以连接为导向。
  • 这意味着两台相互通信的计算机通过互联网保持连接。
  • 但是，HTTP没有。
  • 在客户端提出 HTTP 请求之前，必须向服务器进行新的连接。

三次握手和四次挥手

TCP提供了一种可靠、面向连接、字节流、传输层的服务，采用三次握手建立一个连接。采用4次挥手来关闭一个连接。

作用

三次握手作用：

  1. 确认双方的接受能力、发送能力是否正常。
  2. 指定自己的初始化序列号，为后面的可靠传送做准备。
  3. 如果是 https 协议的话，三次握手这个过程，还会进行数字证书的验证以及加密密钥的生成到。

四次挥手作用：
TCP连接是双向传输的对等的模式，就是说双方都可以同时向对方发送或接收数据。
当有一方要关闭连接时，会发送指令告知对方，我要关闭连接了。

过程

三次握手：

  1. 第一次握手：客户端发送网络包，服务端收到了。这样服务端就能得出结论：客户端的发送能力、服务端的接收能力是正常的。
  2. 第二次握手：服务端发包，客户端收到了。这样客户端就能得出结论：服务端的接收、发送能力，客户端的接收、发送能力是正常的。不过此时服务器并不能确认客户端的接收能力是否正常。
  3. 第三次握手：客户端发包，服务端收到了。这样服务端就能得出结论：客户端的接收、发送能力正常，服务器自己的发送、接收能力也正常。

A：我要传输
B：知道了
A：传输

四次挥手：

  1. 第一次挥手：客户端发送一个 FIN 报文，报文中会指定一个序列号。此时客户端处于FIN_WAIT1状态。
  2. 第二次握手：服务端收到 FIN 之后，会发送 ACK 报文，且把客户端的序列号值 + 1 作为 ACK 报文的序列号值，表明已经收到客户端的报文了，此时服务端处于 CLOSE_WAIT状态。
  3. 第三次挥手：如果服务端也想断开连接了，和客户端的第一次挥手一样，发给 FIN 报文，且指定一个序列号。此时服务端处于 LAST_ACK 的状态。
  4. 第四次挥手：客户端收到 FIN 之后，一样发送一个 ACK 报文作为应答，且把服务端的序列号值 + 1 作为自己 ACK 报文的序列号值，此时客户端处于 TIME_WAIT 状态。需要过一阵子以确保服务端收到自己的 ACK 报文之后才会进入 CLOSED 状态

B：知道了
B：断开连接
A：知道了

级联

相当于把一个端口进行扩展成多个端口，扩展的端口总带宽=级联口的带宽

堆叠

相当于往交换机上增加端口，所有增加的端口跟之前的端口共享交换机的背板带宽

集群
